# Coding Conventions

This project favors simple, readable, and reproducible code.

## Rules

- Use clear variable names.
- Keep functions short.
- Each function should do one thing.
- Avoid unnecessary classes.
- Avoid over-engineering.
- Add comments to explain why a step is needed.
- Do not add steps only for decoration.
- Save generated tables to eports/tables/.
- Save generated figures to eports/figures/.

## Evidence policy

Do not write conclusions before results are available.

Use this phrase when needed:

> No data is available yet to support this conclusion.
